Organization of data

Change of Thinking


Skills of working with SQL databases are not suitable for NoSQL
  • No JOIN
  • Data redundancy instead of normalization
  • Control data from the application, not from the database

Application area

    Not suitable:
  1. if you use JOINs
  2. if you need transactions


    Suitable for:
  1. heterogeneous storage
  2. geodetic storage
  3. storage of logs and statistics
